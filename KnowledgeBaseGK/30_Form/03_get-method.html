📋 Template 03 – Phương thức GET

Mô phỏng Client.html → Server.html · Môn Phát triển Web Kinh doanh · UEL

CLIENT Form gửi bằng method GET

server.html?name=iloveuel&email=iloveuel%40uel.edu.vn&submit=Submit
⚠ Lưu ý bảo mật: Với GET, dữ liệu hiện trên URL Address bar. Không dùng GET để gửi mật khẩu. Kích thước tối đa của URL là 2048 bytes.

SERVER server.html nhận và đọc dữ liệu từ URL

📥 Dữ liệu nhận được:

Name: iloveuel
Email: iloveuel@uel.edu.vn

👆 Nhấn Submit ở trên để xem giá trị thay đổi theo

Code đọc URL params trong server.html:

// ── Cách 1: URLSearchParams (KHUYẾN DÙNG) ─────────────
const params = new URLSearchParams(window.location.search);
// window.location.search = "?name=iloveuel&email=..."

const name  = params.get("name");   // → "iloveuel"
const email = params.get("email");  // → "iloveuel@uel.edu.vn"

// Hiển thị vào DOM
document.getElementById("name").innerText  = name;
document.getElementById("email").innerText = email;

// ── Cách 2: Regex thủ công (cũ hơn) ──────────────────
function getUrlParams() {
  var params = {};
  var qs = window.location.href
    .replace(/[?&]+([^=&]+)=([^&]*)/gi, function(m, key, value) {
      params[key] = value;
    });
  return params;
}
var name  = unescape(getUrlParams()["name"]);
var email = unescape(getUrlParams()["email"]);

⚖ So sánh GET vs POST

Tiêu chí GET POST
Vị trí data Gắn vào URL (?key=val) Trong request body (ẩn)
Giới hạn Max 2048 bytes Không giới hạn
Bảo mật Thấp – thấy trên URL Cao hơn – ẩn trong body
CRUD Read – truy vấn dữ liệu Create – lưu mới dữ liệu
Trang nhận HTML tĩnh cũng dùng được Cần server backend (PHP, Node…)
Bookmark Có thể bookmark URL kết quả Không bookmark được
