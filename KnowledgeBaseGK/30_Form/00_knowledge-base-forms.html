1. Giới thiệu về Form

Form là gì?

Form dùng để thu thập dữ liệu đầu vào của người dùng rồi gửi lên máy chủ (server) để xử lý. Ví dụ điển hình: form đăng nhập Facebook, form tạo tài khoản Google.

Cú pháp cơ bản
<form action="url_xu_ly"
      method="get | post"
      name="ten_form"
      target="_blank | _self">

  <!-- Các phần tử nhập liệu bên trong -->
  <input type="text" name="ten">
  <input type="submit" value="Gửi">

</form>

2. Phương thức GET · POST · PUT · DELETE
CRUD mapping: POST = Create · GET = Read · PUT = Update/Replace · DELETE = Delete

⬆ GET

  • Dữ liệu gắn trực tiếp vào URL sau dấu ?
    Ví dụ: ?name=JAJA&email=a@b.com
  • Giới hạn kích thước: tối đa 2048 bytes
  • Dữ liệu hiển thị trên thanh địa chỉ → không bảo mật
  • Dùng để: tra cứu / tìm kiếm dữ liệu
  • Phù hợp với trang HTML tĩnh (server.html)

⬆ POST

  • Dữ liệu truyền "ngầm" trong body request
  • Không giới hạn kích thước dữ liệu
  • Dữ liệu không hiển thị trên URL → bảo mật hơn
  • Dùng để: lưu mới dữ liệu lên server
  • Cần server-side script (PHP, Node.js…) để xử lý
Đọc tham số GET trong JavaScript
// Cách 1: URLSearchParams (đơn giản, hiện đại)
const params = new URLSearchParams(window.location.search);
const name  = params.get("name");   // lấy giá trị field "name"
const email = params.get("email");  // lấy giá trị field "email"

// Hiển thị ra trang
document.getElementById("name").innerText  = name;
document.getElementById("email").innerText = email;

3. <label> và <input>

<label> Nhãn nhập liệu

  • Tạo tiêu đề cho control trong form
  • for="id_control" — liên kết với control cùng id
  • Nhấp vào label → focus vào control tương ứng
<label for="username">Tên đăng nhập:</label>
<input type="text" id="username">
<!-- for="username" ↔ id="username" phải khớp -->

<input> Thuộc tính chung

  • name — quan trọng nhất, là key khi gửi dữ liệu
  • type — loại ô nhập liệu
  • value — giá trị mặc định
  • placeholder — gợi ý văn bản mờ trong ô
  • required — bắt buộc nhập, không được để trống
  • id — dùng để label liên kết

reset Nút đặt lại tất cả về giá trị mặc định value <input type="reset" value="Xóa">

Phân biệt checkbox vs radio

☑ checkbox — Chọn nhiều

<!-- Mỗi checkbox có name KHÁC nhau -->
<input type="checkbox" name="clip"  value="clip">  ClipTV
<input type="checkbox" name="film"  value="film">  Film+
<input type="checkbox" name="fpt"   value="fpt" checked> FPT Play

→ Mỗi checkbox name khác nhau để nhận nhiều giá trị độc lập

● radio — Chọn một

<!-- Cùng name="gender" → chỉ chọn được 1 -->
<input type="radio" name="gender" value="male"   checked> Nam
<input type="radio" name="gender" value="female"> Nữ

→ Các radio cùng name thì chỉ chọn được 1 trong nhóm đó

5. <select> và <datalist>

<select> Danh sách thả xuống

  • name — tên field
  • multiple — cho phép chọn nhiều (giữ Ctrl/Cmd)
  • size="3" — hiện 3 dòng trực tiếp
<select name="car" id="car">
  <option value="vol">Volvo</option>
  <option value="audi" selected>Audi</option>
  <option value="bmw">BMW</option>
</select>
<!-- selected = tùy chọn mặc định -->

<optgroup> Nhóm tùy chọn

  • label — tên nhóm (không chọn được)
  • disabled — tắt cả tùy chọn trong nhóm
<select name="car">
  <optgroup label="Nhóm 1">
    <option value="vol">Volvo</option>
    <option value="aud">Audi</option>
  </optgroup>
  <optgroup label="Nhóm 2" disabled>
    <option value="bmw">BMW</option>
  </optgroup>
</select>

<datalist> Gợi ý tự động hoàn thành

  • Kết hợp ô nhập text + danh sách gợi ý
  • <input list="id"> liên kết với <datalist id="id">
  • Người dùng vẫn có thể gõ giá trị không có trong list
<input list="browsers" name="browser">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
</datalist>
<!-- list= phải khớp với id= của datalist -->
6. <textarea> · <fieldset> / <legend> · <output>

<textarea> Nhập văn bản nhiều dòng

  • rows="5" — chiều cao (số dòng hiển thị)
  • cols="30" — chiều rộng (số ký tự mỗi hàng)
  • name — tên field gửi lên server
  • Nội dung mặc định đặt giữa 2 thẻ
<textarea name="message"
          rows="5" cols="30"
          placeholder="Nhập nội dung...">
</textarea>

<fieldset> + <legend> Nhóm controls

  • <fieldset> — vẽ đường viền nhóm các input
  • <legend> — tiêu đề cho nhóm (hiện ở viền trên)
<fieldset>
  <legend>Thông tin cá nhân</legend>
  <label>Họ tên:
    <input type="text" name="fullname">
  </label>
</fieldset>

<output> Hiển thị kết quả tính toán

  • for="a b" — tham chiếu các input đầu vào
  • Dùng với event oninput trên form
<form oninput="x.value=parseInt(a.value)
  +parseInt(b.value)">
  <input type="range" id="a" name="a" value="50">+
  <input type="number" id="b" name="b" value="50"> =
  <output name="x" for="a b"></output>
</form>

⚠ Những điểm thường bị sai khi thi / kiểm tra:
1. for của label phải khớp đúng với id của input (không phải name).
2. Radio buttons cùng nhóm phải có name giống nhau; checkbox cùng nhóm thường có name khác nhau.
3. <datalist> liên kết với <input list="..."> qua id, không phải name.
4. GET gắn data vào URL → không dùng cho mật khẩu. POST ẩn data trong body → cần server backend.
